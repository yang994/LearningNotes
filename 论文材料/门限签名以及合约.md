# 门限签名
n个用户中，收集t个签名（t<=n），签名有效。可以理解为将签名拆解为很多份，收集齐门限规定的份数才可以获得真正的签名。
## 多重签名MultiSSig
多个私钥分开保管，n个私钥中有t个参与签名，才可以通过。有多个锁孔的锁
## 秘密共享Secret Sharing
将秘钥分为几个部分，用冗余的方式分开保管。类似纠删码

## 拉格朗日插值法
+ n次曲线f(x)，知道n+1个点，可以解出曲线方程。

## 方案理解
+ user有私钥d，公钥P=d*G。
+ 有n个keeper，设门限为k+1，user选择一条曲线，将d作为曲线上的一个点，在曲线上选择n个点作为keeper的私钥，这些私钥可以通过拉格朗日插值法计算出d。由私钥可以计算出公钥。
+ user将私钥`分别发送`给对应的keeper，将所有公钥公开用于验证。
+ keeper组对某个信息做出签名，签名可以被单独验证，任意拿出k+1个签名，使用user的公钥P进行聚合验证，可以等价于验证user使用私钥进行的签名。

## 想法
+ 只需要拿出门限指定个数的签名就可以通过验证，多于这个数量的签名其实是没有意义的。用门限签名的方法通过验证，无法发现没有做出贡献的keeper，这里需要其他的手段

# 智能合约
运行在区块链网络上的一段代码,如：
```solidity
pragma solidity ^0.4.18;
contract hello {
    string greeting;

    function hello(string _greeting) public {
        greeting = _greeting;
    }

    function say() constant public returns (string) {
        return greeting;
    }
}
```
理解为一个可以被调用的合约实例，其中的方法接收参数，进行计算，返回结果。
## 执行合约的代价 gas
合约中的方法由链上的节点（矿工）执行，调用者需要为执行者支付费用。gas一方面是为链上运算资源支付的报酬，一方面也能防止产生大量无意义的操作导致网络瘫痪。  
支付费用=gas消耗量*gas价格，gas消耗量由调用程序的复杂程度决定，gas价格为调用时设置的，愿意为gas付出的价格  
对于矿工来说，会优先运行gas价格高的请求，这样可以保证利润最大化。gas价格太低，没有节点愿意在第一时间运行调用者的请求，一次合约的调用就会被搁置甚至一直无法被运行。
